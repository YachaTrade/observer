# GIWA Monad 잔재 정리 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** GIWA Sepolia 신규 배포를 막거나 오염시키는 Monad 잔재(DefiLlama 슬러그, migration 주소 상수, env 계약, CI, 배포 템플릿)를 정리한다.

**Architecture:** 런타임 로직은 변경하지 않는다. 상수·설정·시드만 교체한다: DefiLlama coin ref는 env 주도(`DEFILLAMA_CHAIN_SLUG`, 기본 `ethereum`)로, migration의 Monad WMON 주소 DEFAULT/시드는 GIWA WETH predeploy로, env 계약은 `WMON`→`WETH`로 바꾸고 `.env.example`을 신설한다.

**Tech Stack:** Rust 2024 (alloy, sqlx, lazy_static/LazyLock), PostgreSQL migration SQL, GitHub Actions, bash.

**Spec:** `docs/superpowers/specs/2026-07-17-giwa-monad-residue-cleanup-design.md`

## Global Constraints

- WETH predeploy 주소: `0x4200000000000000000000000000000000000006` (EIP-55 체크섬 형태와 동일 — 알파벳 hex 없음)
- Pyth ETH/USD feed: `0xff61491a931112ddf1bd8147cd1b641375f79f5825126d665480874634fd0ace`
- GIWA Sepolia RPC 기본/예시값: `https://sepolia-rpc.giwa.io`
- DefiLlama 슬러그 기본값: `ethereum`
- Migration 교체 규칙: 컬럼 `DEFAULT`·시드 `INSERT`의 Monad 주소만 교체, 백필 `UPDATE`/`WHERE` 리터럴은 보존(빈 GIWA DB에서 no-op인 히스토리 스크립트)
- 건드리지 않기: `migrations/0010_raffle.sql`, `migrations/1000_delete.sql`, `docs/plans/`, `branches/`, `src/db/cache/mod.rs` 등 내부 WMON 개념 주석
- 각 태스크 끝에 커밋. 최종 태스크에서 `cargo fmt --all -- --check`, `cargo clippy -- -D warnings`, 전체 테스트

```rust
/// DefiLlama chain slug for coin refs, env-overridable per deployment.
/// Default "ethereum": GIWA whitelist tokens map to their ethereum-mainnet
/// equivalents via `whitelist_token.price_source_id` (DefiLlama does not
/// index GIWA itself).
static DEFILLAMA_CHAIN_SLUG: std::sync::LazyLock<String> = std::sync::LazyLock::new(|| {
    std::env::var("DEFILLAMA_CHAIN_SLUG").unwrap_or_else(|_| "ethereum".to_string())
});

pub fn coin_ref(token_id: &str) -> String {
    format!("{}:{token_id}", &*DEFILLAMA_CHAIN_SLUG)
}
```

`src/config.rs` 25-37행을 아래로 교체:

```rust
    // WETH address, normalized to EIP-55 checksum form at load time.
    // The env var may be set in any valid casing (lowercase, checksum,
    // etc.); we parse it through `alloy::primitives::Address` (case-
    // insensitive) and re-emit it via `Display`, which writes EIP-55.
    // This guarantees lex-equality with every alloy-derived address
    // downstream regardless of operator env casing.
    pub static ref WNATIVE_ADDRESS: String = {
        let raw = env::var("WETH").expect("WETH must be set");
        let parsed: Address = raw
            .parse()
            .unwrap_or_else(|e| panic!("WETH env var is not a valid EVM address '{}': {}", raw, e));
        parsed.to_string()
    };
```

같은 파일 208행의 panic 메시지 `(e.g. WMON)` → `(e.g. WETH)`.
